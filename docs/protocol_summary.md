# Protocol Summary

The study compared two prompting conditions in an LLM-mediated domain learning task for Requirements Engineering.

## Participants

Eight participants were anonymized as P01 to P08.

The group included different professional profiles, such as developers, analysts, support professionals, interns, and requirements-oriented practitioners.

No real names are required for reproducing the study.

## Domain

The domain was public works monitoring, focusing on two artifacts:

- Measurement Bulletin;
- Construction Monitoring Report.

The domain was selected because it requires understanding relationships among documents, business rules, validation rules, evidence, responsibilities, and ambiguities.

## Conditions

Two experimental conditions were compared:

- Condition A: standard structured prompt.
- Condition B: structured prompt plus anonymized didactic strategy derived from OCEAN/IPIP-NEO profiles.

Both conditions used the same technical domain base and the same task structure.

The only intended difference was the explanatory strategy used in Condition B.

## Procedure

1. Participants completed a pre-test, referred to as K0.
2. OCEAN/IPIP-NEO profiles were converted into anonymized didactic strategies.
3. The controlled technical domain base was prepared.
4. The LLM generated instructional materials for each participant-task-condition combination.
5. Participants received the generated materials.
6. Participants completed post-tests, referred to as K1.
7. The planned artifact production and blind expert review stage was not executed and remains future work.

## Executed stages

- K0 pre-test: executed.
- OCEAN/IPIP-NEO profiling: executed.
- Didactic strategy construction: executed.
- Controlled technical domain base: executed.
- Prompt templates for A and B: executed.
- LLM material generation via API: executed.
- K1 post-tests: executed.

## Non-executed stages

- Participant-produced requirements artifacts.
- Blind expert evaluation of artifacts.
- Knowledge graph condition.

## Corrected aggregate results

- K0 average: 14.00/44 = 31.82%
- K1 average: 41.13/44 = 93.47%
- Learning gain: +61.65 p.p.
- Condition A: 164/176 = 93.18%
- Condition B: 165/176 = 93.75%
- Difference B - A: +0.57 p.p.

## Scope

The protocol supports analysis of initial domain learning.

It does not support conclusions about the quality of participant-produced requirements artifacts, because that stage was not executed.
