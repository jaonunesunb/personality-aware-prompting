# Experimental Design Matrix

This file summarizes the balanced experimental assignment used in the pilot study.

## Conditions

- **Condition A:** standard structured prompt.
- **Condition B:** structured prompt plus anonymized didactic strategy derived from OCEAN/IPIP-NEO profiles.

Both conditions used the same controlled technical domain base, task structure, and mandatory output format.

## Tasks

The study used two domain-learning tasks:

- Measurement Bulletin;
- Construction Monitoring Report.

## Participant assignment

| Participant | Measurement Bulletin | Construction Monitoring Report |
|---|---|---|
| P01 | Condition A | Condition B |
| P02 | Condition B | Condition A |
| P03 | Condition A | Condition B |
| P04 | Condition B | Condition A |
| P05 | Condition A | Condition B |
| P06 | Condition B | Condition A |
| P07 | Condition A | Condition B |
| P08 | Condition B | Condition A |

## Balance

The design was balanced as follows:

| Task | Condition A | Condition B |
|---|---:|---:|
| Measurement Bulletin | 4 participants | 4 participants |
| Construction Monitoring Report | 4 participants | 4 participants |

## Purpose of the design

The purpose of the design was to reduce the risk that one condition would be favored simply because it was associated with one specific artifact.

Each condition was applied to both artifacts across the full participant group.
